# Search — testid contract

| Key                       | data-testid                  | Element                                                            |
|---------------------------|------------------------------|--------------------------------------------------------------------|
| `headerSearchInput`       | `header-search-input`        | Search `<input>` in the storefront header                          |
| `headerSearchSubmit`      | `header-search-submit`       | Magnifying-glass submit button (if present as a separate element)  |
| `headerSearchClear`       | `header-search-clear`        | Inline clear-input "×" button                                      |
| `searchSuggestions`       | `search-suggestions`         | Suggestions panel that opens below the input                       |
| `searchSuggestionItem`    | `search-suggestion-item`     | A single suggestion row                                            |
| `searchSrp`               | `search-srp`                 | Search results page root                                           |
| `searchSrpHeading`        | `search-srp-heading`         | The "results for query" heading / chip                             |
| `searchSrpGrid`           | `search-srp-grid`            | The grid that holds the product cards                               |
| `searchSrpCard`           | `search-srp-card`            | A single result card                                                |
| `searchSrpEmpty`          | `search-srp-empty`           | Empty-state container («نتیجه‌ای یافت نشد»)                          |

Fallback rules: same as `../homepage/testid-mapping.md` — role / structural selectors first, never numeric / price-based anchors.
